📱 Telegram Bot

Role: Primary control interface. Commands in, results out.
Protocol: Bot API long-polling (getUpdates)
Auth: TELEGRAM_BOT_TOKEN + TELEGRAM_CHAT_ID
Slash gate: All commands require / prefix
Retry: Markdown parse fail → auto-retry plain text
output/telegram_bot.py output/telegram_notifier.py api.telegram.org

⏰ Scheduler

Role: Auto-scan at 3 fixed ET slots + manual scan dispatch
Slots: 09:35 Open · 12:45 Midday · 15:00 Pre-Close
Loop: Checks every 10s, fires once per slot per day
Threading: Scan runs in daemon thread
Cancel: threading.Event, checked at batch boundary
scheduler.py Main entry point

🖥️ Mac Mini Hub

Role: Always-on host for bot + TWS/IB Gateway
OS: macOS
Start: START_BOT.sh / START_BOT_BACKGROUND.sh
Python: 3.11+
Future: OpenClaw autonomous agent
START_BOT.sh START_BOT_BACKGROUND.sh

🧠 Scanner Engine

Role: Orchestrate full scan pipeline
Flow: Config → Fetch chain → Greeks → IV filter → Strategy filter → Score → Rank
IV filter: Two layers — min_iv_rank (relative) + min_iv (absolute)
Batch: 30 contracts/batch, 6s wait per batch
Output: List[ScanOpportunity] sorted by score
core/scanner.py core/config.py core/models.py

📊 Scoring & Greeks

Score (0-100): 5 weighted factors
  Ann. Return 30% · IV 20% · Theta Yield 20% · Delta Safety 20% · Liquidity 10%
Greeks: Black-Scholes (scipy.stats.norm), daily theta for sellers
IV Rank: (current - 52w_low) / (52w_high - 52w_low) × 100
Risk-free rate: 5.3% (review periodically)
core/scorer.py core/greeks.py

📋 Strategy Filters

Covered Call: Call options, strike > underlying, delta 0.20–0.35, OTM
Cash-Secured Put: Put options, strike < underlying, delta -0.35 to -0.20, OTM
Shared filters: min_theta, min_iv_rank, min_iv, max_vega, min_premium, min_ann_return
Telegram override: /set strategy cc | csp | both
strategies/covered_call.py strategies/cash_secured_put.py

🏦 IBKR (TWS / IB Gateway)

Role: Option chain data, IV history, contract qualification
Library: ib_insync (async TWS API)
Ports: 7497 (TWS paper) · 4002 (Gateway paper) · 7496 · 4001
Data type: Market open → Type 3 (delayed) · Closed → Type 4 (frozen)
Quirk: Delayed data returns OI=0 — keep min_oi=0
Error 10197: Competing sessions (phone/browser) block data
data/ibkr_fetcher.py localhost:7497/4002

Data Flow Routes

📡 Scan Pipeline: Scheduler triggers → Yahoo price → IBKR chain fetch → Greeks calc → IV + strategy filter → Score & rank → Supabase write → Telegram notify
📱 Telegram cmd: User sends /command → Bot long-poll receives → Slash gate check → Command router → Handler executes → Reply sent
🔄 Trade workflow: Scan (auto/manual) → pending (Supabase) → /star N → starred → /approve N → approved → Place in IBKR → /placed N → trade_log
🌐 Dashboard: User visits netlify → React SPA loads → Supabase query → Display candidates/trades
🤖 AI query: /askclaude question → Anthropic API (claude-sonnet-4-6) → Reply to Telegram
🦙 AI query: /askllama question → OpenRouter API (llama-3.3-70b) → Reply to Telegram

Diagnostic Quick Reference

❌ No scan results: Check /config filters → Try: /set min_iv 0.30 → Check IBKR connection (/health) → Check Yahoo price works
❌ IBKR Error 10197: Close all other TWS sessions → Phone, browser, other computers → Only Mac Mini should be connected
❌ Telegram 400: Markdown parse error → Bot auto-retries as plain text → Check for _ outside ``` blocks
❌ Supabase fail: Non-fatal (scan still delivers) → Check .env SUPABASE_URL/KEY → /health shows status
❌ Scan stuck: /cancelscan (1st try) → /cancelscan again → force reset → If still stuck: restart bot
